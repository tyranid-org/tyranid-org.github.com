Information Management System (IMS)

vs.

Database Management System (DBMS)

This problem is best solved in a layer outside of a DBMS because the ultimate answer spans multiple types of DBMSes, search engines, applications, etc.

Philosophy

  • data-first framework (tyranid) vs. ui-first framework (angular, ember, meteor, etc.)
  • pragmatism: incremental building of features as needed

The Present

Updated documentation

http://tyranid.org

Client Support

  • Client-side versions of classes are generated automatically from metadata.
  • Supports client-side, server-side, and isomorphic properties and methods
  • Automatic route handlers to enable methods like $save() on the client.
more information

Link Constraints

var User = new Tyr.Collection({
  ...
  fields: {
    ...
    // Managers have to be in the same organization as the user.
    manager:   { link: 'user', where: { organizationId: $this.organizationId } },
    ...
  }
});
more information

Paths and Fields

  • Field object allows an API at the Field level. For example, Field.labels() method provides automated type-ahead dropdown support.
  • Available on client making it easier to write automated UI.
  • Paths make it easier to work with form UI controls.

Query Optimization

  • Using logging, keep a log of queries against mongo and suggest/manage indices that should be created/removed.
  • Cross referencing of query timings with route logging is powerful ... i.e. "What are the top 10 slowest high traffic api links and why are they slow?"
  • Ability to create data warehouses/summary collections automatically provides enhanced optimization possibilities ... if a query is super high traffic and can be queried lots of different ways might be better to create a summary table on it than 15 different indices.
  • If database deployment management is integrated, solution might also be to allocate a larger cluster, etc.
