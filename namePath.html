NamePath

NamePaths are returned by collection.parsePath(path). They are used to make writing generic metadata-based code easier.

NamePath is also able to deal with denormalized paths as well.

See paths for more information.

get(obj: object): any

This walks the obj according to this path and returns the values at this path.

name: string

This contains the full, unparsed path that was passed into the constructor.

path: string[]

This contains the path split up into individual strings for steps on the path.

'name'[ 'name' ]
'name.first'[ 'name', 'first' ]
'organization_.owner_.name.first'[ 'organization_', 'owner_', 'name', 'first' ]
pathFields: Field[]

This array contains the Field for each step along the path. This is a parallel array to path.

pathName(idx: integer): string

This returns a verbose description of the path at position idx along the path. This is useful for error messages in parsing.

tail(): Field

This is a shortcut for pathFields[pathFields.length-1]. Often you are only interested in the end of the path which has the Type of the path you are working with.

uniq(obj: object): any[]

This walks the obj according to this path and returns the uniq values at this path it finds. (There can be multiple values if the path points to an array.)
